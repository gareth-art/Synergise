## FIX 2: SESSION TIMEOUT DURING ONBOARDING

Users are being kicked back to login with "Your session expired" message while completing onboarding. This must never happen.

### Fix A: Extend session maxAge in server config
In app.ts or server/index.ts, find the express-session config and update:

```ts
cookie: {
  secure: process.env.NODE_ENV === 'production',
  httpOnly: true,
  sameSite: 'lax',
  maxAge: 30 * 24 * 60 * 60 * 1000 // 30 days — users should stay logged in
}
```

30 days is standard for SaaS applications. Users should not be logged out unless they explicitly sign out.

### Fix B: Add session keepalive ping
In the dashboard layout AND the onboarding page, add a silent background ping every 4 minutes to keep the session alive:

```tsx
useEffect(() => {
  const ping = setInterval(async () => {
    await fetch('/api/auth/me', { credentials: 'include' }).catch(() => {});
  }, 4 * 60 * 1000); // every 4 minutes
  return () => clearInterval(ping);
}, []);
```

Add this to:
- Onboarding.tsx (so session stays alive while user is filling in forms)
- DashboardLayout.tsx (so session stays alive during normal use)

### Fix C: Handle session expiry gracefully — never interrupt onboarding
In Onboarding.tsx handleComplete function, if the POST /api/onboarding returns 401:
- Do NOT redirect to login immediately
- First attempt to re-authenticate silently: GET /api/auth/me with credentials: include
- If that returns 200 (session actually still valid, was a transient error): retry the POST /api/onboarding once
- If GET /api/auth/me returns 401 (session genuinely expired): show this message inline in the card (do not redirect):

```tsx
<div className="bg-amber-50 border border-amber-200 rounded-xl p-4 text-sm text-amber-800">
  <strong>Your session expired</strong> while you were filling in the form. 
  Your answers have been saved. 
  <button onClick={() => setLocation('/login?prefill=true')} className="underline ml-1">
    Sign in again to continue
  </button> — you won't need to redo this step.
</div>
```

Do not auto-redirect. Let the user choose when to re-authenticate.

### Fix D: Preserve onboarding form data across session expiry
In Onboarding.tsx, save form state to sessionStorage on every field change:

```tsx
// Save to sessionStorage whenever any field changes
useEffect(() => {
  sessionStorage.setItem('onboarding_draft', JSON.stringify({
    businessName,
    industry,
    region,
    revenueStage,
    currentStep
  }));
}, [businessName, industry, region, revenueStage, currentStep]);

// Restore from sessionStorage on mount
useEffect(() => {
  const draft = sessionStorage.getItem('onboarding_draft');
  if (draft) {
    try {
      const parsed = JSON.parse(draft);
      if (parsed.businessName) setBusinessName(parsed.businessName);
      if (parsed.industry) setIndustry(parsed.industry);
      if (parsed.region) setRegion(parsed.region);
      if (parsed.revenueStage) setRevenueStage(parsed.revenueStage);
      if (parsed.currentStep) setCurrentStep(parsed.currentStep);
    } catch {}
  }
}, []);

// Clear draft after successful submission
// In handleComplete success path:
sessionStorage.removeItem('onboarding_draft');
```

This means even if the session expires and the user signs back in, their onboarding answers are still there when they return to /onboarding.
