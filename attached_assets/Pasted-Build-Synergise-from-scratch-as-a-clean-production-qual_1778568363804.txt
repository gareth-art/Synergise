## DATABASE SCHEMA

Create these tables via Drizzle:

### users
- id (serial primary key)
- username (text, unique)
- email (text, unique)
- password (text, hashed with bcrypt)
- fullName (text)
- subscriptionTier: 'trial' | 'professional' (default: 'trial')
- trialStartDate (timestamp)
- createdAt (timestamp)

### onboarding_profiles
- id (serial primary key)
- userId (integer, foreign key → users.id)
- businessName (text)
- industry (text) — values: wellness, consumer-products, membership, professional-services, fnb, ecommerce, saas, other
- region (text) — values: sg, sea, apac, global, north-america, europe
- revenueStage (text) — values: pre-revenue, under-500k, 500k-2m, over-2m
- onboardingCompletedAt (timestamp)

### financial_models
- id (serial primary key)
- userId (integer, foreign key → users.id)
- industry (text)
- modelName (text)
- inputs (jsonb) — flexible schema for industry-specific inputs
- outputs (jsonb) — calculated results
- createdAt (timestamp)
- updatedAt (timestamp)

### management_accounts
- id (serial primary key)
- userId (integer, foreign key → users.id)
- period (text) — e.g. "2026-Q1"
- revenue (numeric)
- cogs (numeric)
- grossProfit (numeric)
- grossMarginPct (numeric)
- operatingExpenses (jsonb)
- ebitda (numeric)
- netProfit (numeric)
- createdAt (timestamp)

### kpi_benchmarks
- id (serial primary key)
- industry (text)
- region (text)
- metric (text)
- p10 (numeric), p25 (numeric), p50 (numeric), p75 (numeric), p90 (numeric)
- updatedAt (timestamp)

### admin_segments (for internal analytics)
- id (serial primary key)
- userId (integer)
- industry (text)
- region (text)
- revenueStage (text)
- subscriptionTier (text)
- signupDate (timestamp)

## AUTHENTICATION

- Use Passport.js with local strategy
- Passwords hashed with bcrypt (10 rounds)
- Sessions via express-session + memorystore
- Protected routes: redirect to /login if not authenticated
- After login/signup: check if onboarding_profile exists → if not, redirect to /onboarding → else /dashboard
- POST /api/auth/signup — creates user, logs in, returns user object
- POST /api/auth/login — authenticates, returns user object
- POST /api/auth/logout — destroys session
- GET /api/auth/me — returns current user or 401
