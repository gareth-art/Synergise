## FIX 5: ONBOARDING GUARD — DON'T REPEAT COMPLETED ONBOARDING

The app must never send a user back to /onboarding if they have already completed it.

Create or update a useOnboardingStatus hook:
```tsx
export function useOnboardingStatus() {
  const { data: profile, isLoading } = useQuery({
    queryKey: ['/api/onboarding'],
    queryFn: async () => {
      const res = await fetch('/api/onboarding', { credentials: 'include' });
      if (res.status === 404) return null;
      if (!res.ok) throw new Error('Failed to fetch onboarding status');
      return res.json();
    },
    staleTime: 5 * 60 * 1000, // cache for 5 minutes
    retry: false
  });
  return { 
    isComplete: !!profile, 
    isLoading,
    profile 
  };
}
```

In the /onboarding route component (ProtectedRoute or the onboarding page itself):
```tsx
const { isComplete, isLoading } = useOnboardingStatus();

if (isLoading) return <LoadingSpinner />;
if (isComplete) {
  setLocation('/dashboard');
  return null;
}
// render onboarding wizard
```

In the dashboard ProtectedRoute:
- If user is authenticated AND onboarding is NOT complete → redirect to /onboarding
- If user is authenticated AND onboarding IS complete → render dashboard
- While isLoading → show full-page loading spinner (NOT a redirect)
